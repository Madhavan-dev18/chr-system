# 🏥 CHR-System (Clinical Health Records System)

A state-of-the-art, hyper-secure, clinical health record management platform designed for modern medical facilities. CHR-System features strict multi-tenant data isolation, granular Role-Based Access Control (RBAC), and AES-256-GCM encryption for all sensitive Protected Health Information (PHI).

## 🚀 Tech Stack

- **Framework**: [Next.js 15](https://nextjs.org/) (App Router, Edge Middleware)
- **Language**: TypeScript (`strict: true`)
- **Database**: PostgreSQL via [Supabase](https://supabase.com/)
- **ORM**: [Prisma 6](https://www.prisma.io/)
- **API**: [tRPC](https://trpc.io/) (End-to-end typesafe)
- **Authentication**: [NextAuth v5](https://authjs.dev/) (JWT)
- **Caching & Rate Limiting**: [Upstash Redis](https://upstash.com/)
- **AI Integration**: Google [Gemini 1.5 Flash](https://aistudio.google.com/)
- **Styling**: TailwindCSS (Custom Neumorphic Design System)

## 🛡️ Security Architecture (OWASP Hardened)

CHR-System was built from the ground up with healthcare compliance (e.g., HIPAA) and OWASP Top 10 in mind:

1. **Edge-Level Rate Limiting**: All API routes and tRPC endpoints are throttled via `@upstash/ratelimit` (sliding window algorithm) at the Next.js edge to prevent DDoS and brute-forcing.
2. **Strict Content Security Policy (CSP)**: `next.config.ts` enforces a rigid CSP to mitigate XSS attacks.
3. **Multi-Tenant Isolation**: Database queries are strictly gated behind `clinicScopedProcedure`, guaranteeing data cannot leak across clinic boundaries.
4. **PHI Encryption**: All medical records are encrypted at rest using AES-256-GCM before ever touching the database.
5. **Input Validation**: `zod` strictly sanitizes all incoming API requests.
6. **Robust RBAC**: Granular roles (`ADMIN`, `DOCTOR`, `NURSE`, `PATIENT`, `RECEPTIONIST`, `LAB_TECH`) enforced simultaneously at the Next.js middleware edge and tRPC server.

## 🛠️ Local Development Setup

### 1. Prerequisites
- Node.js 20+
- Accounts for Supabase (Free Tier), Upstash (Free Tier), and Google AI Studio.

### 2. Installation
```bash
git clone https://github.com/Madhavan-dev18/chr-system.git
cd chr-system
npm install
```

### 3. Environment Configuration
Copy the environment template to your Next.js application:
```bash
cp .env.example apps/web/.env
```
Open `apps/web/.env` and insert your actual API keys for Supabase, Upstash, and Gemini.

### 4. Database Setup
Push the schema to your Supabase PostgreSQL instance and generate the Prisma Client:
```bash
npx prisma db push -w packages/db
```

Seed the database with the root `ADMIN` user:
```bash
npx prisma db seed -w packages/db
```

### 5. Start the Development Server
```bash
npm run dev -w apps/web
```
Navigate to `http://localhost:3000` to log in using the credentials generated by the seed script.
